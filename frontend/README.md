# ELID Frontend - Device Management UI

A modern React + TypeScript frontend for the ELID Device Management System.

## Features

- Create and manage devices (Access Controllers, Face Recognition Readers, ANPR)
- Activate/Deactivate devices with real-time status updates
- View transactions generated by active devices
- Auto-refreshing transaction list
- Responsive design with modern UI

## Prerequisites

- Node.js (v18 or higher)
- npm or yarn
- Backend API running (default: http://localhost:8000)

## Setup

1. Install dependencies:
```bash
npm install
```

2. Configure environment variables:
```bash
cp .env.example .env
```

Edit `.env` and set the API URL if different from default:
```
VITE_API_BASE_URL=http://localhost:8000
```

## Development

Start the development server:
```bash
npm run dev
```

The application will be available at `http://localhost:5173`

## Build for Production

Build the application:
```bash
npm run build
```

Preview the production build:
```bash
npm run preview
```

## Usage

### Creating a Device

1. Navigate to the "Devices" tab
2. Fill in the device form:
   - Device Name
   - Device Type (Access Controller, Face Recognition Reader, or ANPR)
   - IP Address
3. Click "Create Device"

### Activating/Deactivating Devices

1. Find the device in the device list
2. Click the "Activate" button to start generating transactions
3. Click "Deactivate" to stop the device

### Viewing Transactions

1. Navigate to the "Transactions" tab
2. Transactions from active devices will appear automatically
3. Toggle "Auto-refresh" to enable/disable real-time updates

## Technologies Used

- React 19
- TypeScript
- Vite
- CSS3 (Custom styling, no UI frameworks)

## API Endpoints Used

- `GET /devices/list` - List all devices
- `POST /devices/create` - Create a new device
- `PUT /devices/{device_id}` - Toggle device status
- `GET /transactions/list` - List transactions
- `GET /devices/workers/status` - Get active worker count
